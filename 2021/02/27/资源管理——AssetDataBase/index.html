资源管理——AssetDataBase

AssetDatabase是一个对资源进行读写等操作的编辑器类接口，位于工程目录的Assets/Editor文件夹下，需要引入UnityEditor命名空间和宏管理。

#if UNITY_EDITOR
using UnityEditor; //手机上没有UnityEditor文件
#endif

  • AssetDatabase常用的资源操作

    1. 加载资源

      加载类型|方法
      ——-|——-
      加载单个Asset|UnityEngine.Object LoadAssetAtPath(string assetPath, Type type);
      加载路径下优先级最高Asset(一般是第一个)|LoadMainAssetAtPath(string assetPath);
      加载路径下所有Asset|LoadAllAssetsAtPath(string assetPath);
      加载路径下所有sub Asset|LoadAllAssetRepresentationsAtPath(string assetPath);

       Material mat = AssetDatabase.LoadAssetAtPath("Assets/red.mat", typeof(Material)) as Material;
 //或者
 Material mat = AssetDatabase.LoadAssetAtPath<Material>("Assets/red.mat");

    2. 卸载资源

      Resources.UnloadAsset(Object asset);

       Resources.UnloadAsset(mat);//居然不是AssetDatabase下的方法

    3. 导入资源

      ImportAsset(string path, ImportAssetOptions options);

      通常Unity只在需要时自动导入已拖放至该工程的资源，但也可以通过脚本的方式下导入这些资源

      ImportAssetOptions导入资源选项|描述
      ——-|——-
      Default|默认导入选项。
      ForceUpdate|用户启动资源导入。
      ForceSynchronousImport|同步导入所有资源。
      TryFastReimportFromMetaData|允许快速从元数据重新导入。
      ImportRecursive|当一个文件夹被导入，导入所有内容。

       AssetDatabase.ImportAsset("Assets/Textures/texture.jpg", ImportAssetOptions.Default);

    4. 创建资源

      CreateAsset(UnityEngine.Object asset, string path);

      注意Unity中不允许对单个资源同时创建两份，因此CreateAsset前必须保证资源是一个新的实例。创建结束后刷新则可以在path下看到新创建的Asset。

       Material mat2 = Object.Instantiate<Material>(mat);
 AssetDatabase.CreateAsset(mat2, "Assets/red2.mat");
 AssetDatabase.Refresh();

    5. 修改并保存一个已有的Asset

       //修改了mat的颜色
 mat.color = Color.white;
 //标记为已更改的资源,但不会立即保存到磁盘
 EditorUtility.SetDirty(mat);
 //保存到磁盘
 AssetDatabase.SaveAssets();

  • 其他函数汇总

    静态函数|描述
    ——|——
    AddObjectToAsset|添加对象到Asset
    AssetPathToGUID|获得Asset的GUID
    ClearLabels|删除所有Asset标签
    Contains|当对象是一个资源(在资源文件夹中的文件)时返回true，否则返回false(例如场景中的对象或者在运行时生成的对象)。
    CopyAsset|复制Asset
    CreateAsset|创建Asset
    CreateFolder|创建文件夹。
    DeleteAsset|删除Asset
    ExportPackage|导出包
    FindAssets|查找Asset
    GenerateUniqueAssetPath|创建唯一Asset路径
    GetAllAssetBundleNames|获得Asset数据库的所有资源包名称
    GetAssetDependencyHash|获得Asset所有依赖关系的哈希值
    GetAssetOrScenePath|获得资源或场景路径
    GetAssetPath|获得Asset路径
    GetAssetPathFromTextMetaFilePath|获得根据“.meta”文件获得Asset路径
    GetAssetPathsFromAssetBundle|获得从给定assetBundle名获得路径
    GetAssetPathsFromAssetBundleAndAssetName|获得从给定assetBundle名和Asset名获得路径
    GetCachedIcon|获得Asset图标
    GetDependencies|获得依赖关系的路径列表
    GetLabels|获得资源标签
    GetSubFolders|获得子文件夹
    GetTextMetaFilePathFromAssetPath|获取与Asset关联的“.meta”文件
    GetUnusedAssetBundleNames|获得所有未使用的资源包的名字
    GUIDToAssetPath|通过GUID返回Asset路径
    ImportAsset|导入Asset。
    ImportPackage|导入包
    IsForeignAsset|是否是外部Asset
    IsMainAsset|是否是主要Asset吗
    IsNativeAsset|是否是本地Asset吗
    IsOpenForEdit|用来进行编辑模式的版本控制，为真则Asset可以编辑，否则不能编辑
    IsSubAsset|Asset是另一种Asset的子对象么？
    IsValidFolder|给定一个文件夹的绝对路径，如果它存在，返回真，否则为假
    LoadAllAssetRepresentationsAtPath|返回指定路径下所有Asset的陈述
    LoadAllAssetsAtPath|获得Asset包含的所有对象
    LoadAssetAtPath|获得Asset内指定类型的子对象
    LoadMainAssetAtPath|返回Asset对象组中第一个对象
    MoveAsset|移动Asset
    MoveAssetToTrash|将Asset移动到垃圾桶
    OpenAsset|用相关应用程序打开Asset
    Refresh|刷新
    RemoveAssetBundleName|从Asset数据库删除assetBundle名字
    RemoveUnusedAssetBundleNames|在Asset数据库中删除未使用的assetBundle名字
    RenameAsset|重命名Asset
    SaveAssets|保存所有改变的Asset到磁盘
    SetLabels|替换Asset上的标签列表
    StartAssetEditing|开始Asset导入
    StopAssetEditing|停止Asset导入
    ValidateMoveAsset|检查一个Asset文件是否可以从一个文件夹移动到另一个文件夹。（不实际移动文件）
    WriteImportSettingsIfDirty|将导入设置写入磁盘
