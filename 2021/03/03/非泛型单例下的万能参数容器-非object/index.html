非泛型单例下的万能参数容器(非object)

最近在项目中遇到的问题。

已有以下两个类，一个工厂类负责根据泛型进行生产；一个单例基类。现在的需求是：继承单例基类来实现一个工厂管理器（FactoryMgr）。

//泛型工厂
public class Factory<T>{}
//单例
public class Singleton<T> where T : new()
{
    
    private static readonly Lazy<T> _instance = new Lazy<T>(() => new());
    public static T Instance { get { return _instance.Value; } }
    public static string type{ get => typeof(T).ToString(); }
}

FactoryMgr中需要一个容器用于存储工厂，问题就在于这个容器。因为Factory<T>中T为任意类型，因此首先想到的是用object去装，如下

public class  FactoryMgr : Singleton<FactoryMgr>
{
    //委托列表
    public List<Factory<object>> list = new();

    //为list添加委托
    public void AddFactory(Factory<object> factory)
    {
        list.Add(factory);
    } 
}

但用object必然避免不了装箱和拆箱带来的问题，因此想到了用泛型，但是由于FactoryMgr是非泛型单例，所以List<T>这种操作肯定是不行的。由此我想到了利用里氏替换原则，用一个接口IPublicFactory代替object，再用一个继承了IPublicFactory接口的PublicFactory<T>类去装Factory<T>就可以达到效果。代码如下：

public interface IPublicFactory {}
public class PublicFactory<T> : IPublicFactory
{
    public Factory<T> factory;
}

public class FactoryMgr: Singleton<FactoryMgr>
{
    //工厂列表
    public List<IPublicFactory> factorylist = new();

    //添加工厂到列表
    public void AddFactory<T>(Action<T> action)
    {
        factorylist.Add(new PublicFactory<T>());
    }
}

当然这样也有些不足点，因为每次都要new一个PublicFactory<T>占用堆内存。但是相对于装箱拆箱的性能损耗来说可能更好一些。
