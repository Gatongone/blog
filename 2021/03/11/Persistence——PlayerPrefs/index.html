Persistence——PlayerPrefs

Unity中提供了PlayerPrefs公共类用于存储玩家数据，支持int、float、string三种类型数据的存储，这类似于以键值对的形式进行的存储。

  • 相关函数

    • 将数据存入内存中

      当使用Set方法时只会将数据存入内存，而并非磁盘中。也就是说如果游戏在运行时崩溃，将导致数据丢失。

      PS：使用相同的键名将会导致数据覆盖，即使类型不一样。

      PlayerPrefs.SetString("PlayerName", "关东关");
PlayerPrefs.SetInt("PlayerHP", 100);
PlayerPrefs.SetFloat("PlayerATT", 73.52f);
PlayerPrefs.SetInt("PlayerOCD", OCD? 1 : 0 );

    • 将数据保存到磁盘

      PlayerPrefs.Save();

    • 读取数据

      优先从内存中读取，若内存中没有则从磁盘中读取。

      若读取数据失败（比如无法找到键名），则返回默认值：

      int:0、float:0、string:null

      string name = PlayerPrefs.GetString("PlayerName");
int HP = PlayerPrefs.GetInt("HP");
float ATT = PlayerPrefs.GetFloat("ATT");
//参数二为查找失败时返回的默认值
bool OCD = PlayerPrefs.GetInt("PlayerOCD",0) == 0? false : true;

    • 判断数据是否存在

      if( PlayerPrefs.HasKey("PlayerHP") )
{
    //do something
}

    • 删除数据

      //删除某键及数据
PlayerPrefs.DeleteKey("PlayerName");
//删除所有键及数据
PlayerPrefs.DeleteAll();

  • 数据存储位置

    打开Unity中 Edit → Project Settings → Player 得到公司名称和产品名称（一般是项目名称）

    • Windows

      • 打开注册表：regedit 找到 HKEY_CURRENT_USER\Software\Unity\UnityEditor\[公司名称]\[产品名称]

    • Mac OS

      • ~/Library/Preferences/unity.[公司名称].[产品名称].plist

    • Linux

      • ~/.config/unity3d/[公司名称]/[产品名称]

    • Android

      • /data/data/[包名]/shared_prefs/pkg-name.xml

    • IOS

      • ~/Library/Preferences/[应用ID].plist
