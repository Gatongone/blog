资源管理——AssetBundle

开发人员可以通过AssetBundle将一个资源分到多个包或者将多个资源打到一个包内，并且可以实时地从任意位置加载资源包。Asset在打包时维护一个指向它所依赖包的id。相较于Resources,AssetBundle能自己控制资源的精确管理，包括资源的引用计数。更重要的一点，在游戏发布后，可以从服务器或本地中动态地加载资源包对资源进行更新（热更新），这是Resources所不能做到的。

  • AssetBundle常用的资源操作

    1. Asset打包

      • 通过脚本自定义编辑器打包工具

        public class Build
  {
      
      [MenuItem("Publish/BuildAssetBundle")]
      public static void BuildBundle()
      {
          //新建一个包
          AssetBundleBuild abb= new AssetBundleBuild();
          abb.assetBundleName = "Test.unity3d";//包名
          abb.assetNames = new string[] { "Assets/cube.prefab" };//要打包的Asset路径（项目目录下）

          //把包放到一个包组,因为BuildPipeline可以一次打多个包
          List<AssetBundleBuild> builds = new List<AssetBundleBuild>();
          builds.Add(abb);

          //开始打包
          BuildPipeline.BuildAssetBundles(
              "Assets/Bundle",//打包后生成的路径，可以是任意路径不一定在工程目录下
              builds.ToArray(),//要打的包组
              BuildAssetBundleOptions.CollectDependencies,//打包的选项
              BuildTarget.StandaloneWindows//打包的目标平台
          );
      }
  }
        打包选项BuildAssetBundleOptions 描述
        None 无选项
        UncompressedAssetBundle 打包assetBundle时，不压缩数据（意味着assetbundle会更大）
        CollectDependencies 包括所有依赖关系
        CompleteAssets 强制包含整个资源
        DisableWriteTypeTree 在AssetBundle中不包含类型信息，发布web平台时，不能使用该项
        DeterministicAssetBundle 使每个Object具有唯一不变的hash id，可用于增量式发布AssetBoundle
        ForceRebuildAssetBundle 强制重新build所有的AssetBoundle
        IgnoreTypeTreeChanges 执行增量检测时，忽略TypeTree的变化。
        AppendHashToAssetBundleName 将hash加到AssetBoundle名中
        ChunkBasedCompression 创建AssetBundle时使用LZ4压缩。默认情况是Lzma格式，下载AssetBoundle后立即解压。
        StrictMode 如果生成过程中报告了任何错误，则不会build成功
        DryRunBuild 运行时检查
        DisableLoadAssetByFileName 资源包名禁用LoadAsset
        DisableLoadAssetByFileNameWithExtension 资源包拓展名禁用LoadAsset

      • 官方提供的打包工具Asset Bundle Browser

        首先需要在GitHub上下载Asset Bundle Browser到项目。

        导入成功后可以在unitywindows菜单下看到Asset Bundle Browser选项。点击后则可以打开打包窗口AssetBundles，其中一共有三个页签Configure（配置）、Build（打包）、Inspect（检视）。

        打开一个Asset，在Inspector面板最下方可以看到AssetBundle其中左边的菜单是包名，右边的菜单是包名二级引用（可以不填）

        在AssetBundle选择new并起名以此声明为需要被打包的资源，回到AssetBundles窗口,可以在Configure页签中看到需要被打包的资源。

        进入Build页签，点击Build则根据Output Path生成AB包到路径。

        这里重点说一下Build页签下的参数

        • BuildTarget：要打包的目标平台
        • OutPut Path：打包后生成的路径
        • Clear Folders：是否清空文件夹，重新打包（删除不存在的包）,资源较大时打包较久
        • Copy To StreamingAssets：是否拷贝到StreamingAssets文件夹（该文件夹生成到Asset文件夹下，可以在project面板中看到，在某些平台上是只读文件夹）
        • Compression压缩选项：
          • NoCompression：不压缩，解压快，但包比较大
          • LZMA：压缩最小，但是解压最慢，不支持选择性解压（即使只加载一个资源也要解压全部）
          • LZ4：相对于LZMA来说要大一些，可以选择性解压（推荐）
        • 打包选项：
          • Exclude Type Information：在资源包中不包含资源的类型信息
          • Force Rebuild：重新打包时需要重新构建包，但不会删除不存在的包
          • Ignore Type Tree Changes：执行增量检测时，忽略TypeTree的变化。
          • Append Hash：将文件以哈希值的形式添加值资源包
          • Strict Mode：严格模式，如果打包报错，则打包失败
          • Dry Run Build：运行时检查

    2. 资源加载

      注意不能重复加载同一个AB包

      
//1.同步从本地文件加载（可以直接加载AB包）
LoadFromFile(string path)
{
    //加载 AB包
    AssetBundle bundle = AssetBundle.LoadFromFile(path);
    //加载 AB包 中的 资源
    var obj = bundle.LoadAsset<GameObject>("MyObject");
    Instantiate(obj);
}

//2.异步从本地文件加载
LoadFromFileAsync(string path)
{   
    //创建一个 AB包请求 从 本地文件 中返回
    AssetBundleCreateRequest request = AssetBundle.LoadFromFileAsync(path);
    yield return request;

    //从AB包创建请求中 加载 AB包
    AssetBundle bundle = request.assetBundle;

    //异步加载 AB包 中的 资源
    var obj = bundle.LoadAssetAsync<GameObject>("MyObject");
    Instantiate(obj);
}

//3.异步从内存中加载
IEnumerator LoadFromMemoryAsync(string path)
{
    //创建一个 AB包请求 从 内存 中返回
    AssetBundleCreateRequest request = AssetBundle.LoadFromMemoryAsync(File.ReadAllBytes(path));
    yield return request;

    //从AB包创建请求中 加载 AB包
    AssetBundle bundle = request.assetBundle;

    ////异步加载 AB 包中的 资源
    var obj = bundle.LoadFromMemoryAsync<GameObject>("MyObject");
    Instantiate(obj);
}

//4.异步从www中加载

//5.异步从服务器下载
IEnumerator LoadFromWebAsync(string path)
{
    //等待一个服务器请求
    UnityWebRequest request = UnityWebRequest.GetAssetBundle(path);
    yield return request.Send();

    //从 AssetBundle 下载的句柄 中获取 request 中的内容（AB包）
    AssetBundle bundle = DownloadHandlerAssetBundle.GetContent(request);

    //异步加载 AB 包中的 资源
    var obj = (GameObject) bundle.LoadAssetAsync("MyObject");
    Instantiate(obj);
}

    4. 通过主包补全包的依赖信息

      若依赖资源（材质啥的）和预制资源不在同一个AssetBundle内，如果不同时加载这两种包则会出现资源丢失。可以通过主包中manifest获得某个包的依赖信息，并将其添加到包内。

      //path 为 包所在文件夹路径
//mainbundelname 为 主包名
//bundlename 为 需要添加依赖的 包名
public void AddDependenciesBundle(string path, string mainbundelname , string bundlename)
{
    //加载 主包
    AssetBundle bundleMain = AssetBundle.LoadFromFile(path+"/"+mainbundelname);

    //获取 主包描述文件
    AssetBundleManifest maninfest = bundleMain.LoadAsset<AssetBundleManifest>("AssetBundleManifest");

    //获取 主包描述文件 中 所有依赖信息(一般是所依赖的包名)
    string[] infos = maninfest.GetAllDependencies(bundlename);

    //遍历 依赖信息 加载 依赖包
    foreach (var item in infos)
    {
        AssetBundle.LoadFromFile(path + "/" + item);
    }
}

  • 资源优化的建议

    • 预先制定详细合理的美术规划：目标机型什么样的，根据目标机型的配置，规定内存上限来制定场景、特效、UI等方案；
    • 最好在打包AssetBundle时就管理资源间的依赖；
    • 加载Asset时通过引用计数等方式管理资源的卸载；
    • 压缩贴图（移动端最好压缩成ETC/pvrtc）、模型、动作
    • 在切换场景或定时地进行GC
    • 尽量防止资源被长期占用
    • 注意代码造成频繁触发的C#GC（比如在update中new一些比较大的Object，或多次在堆栈中分配资源）
    • 擅于利用分析工具：
      • PC：profilor
      • Android：adb
      • IOS：instruments
