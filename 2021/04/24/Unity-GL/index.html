Unity.GL

Unity.GL是Unity提供的低级别图形库，能发出类似于OpenGL即时模式的渲染指令，一般用于操作活动变换矩阵。当只需要绘制线条或三角形且不想处理网格时一般会使用此类（当然在一般情况下都应该使用 Graphics.DrawMesh 或 CommandBuffer而不使用即时模式）。

  • 如何使用

    • 使用范围

      如果在Update()中调用GL，这将会在渲染摄像机前执行，这很有可能会由于摄像机清除屏幕而导致GL绘制对象不可见。

      一般用于GL绘制的函数：

      OnPostRender()：仅在脚本挂载到相机上才有效，在相机完成场景渲染后调用。

      OnRenderImage：仅在脚本挂载到相机上才有效，在相机完成所有渲染后调用，一般用于修改相机的最终图像（创建全屏后处理效果）。注意若希望在URP中创建自定义全屏效果，应该使用ScriptableRenderPass；若希望在HDRP中使用自定义全屏效果，应该使用 Fullscreen Custom Pass。

      OnRenderObject：可以不挂在相机上，在渲染物体后调用。

    • 具体用法

      以OnPostRender()为例：

      这里用到的几个概念：

      材质通道：就是类似于3DsMax中的那个材质ID通道，比如如果几个物体都用了一个材质，此时如果修改材质，那么所有的物体材质都会被修改。如果希望只是希望修改某个物体的材质，那么只需要把该物体的ID设置为1，并修改该材质的的ID为1的通道材质即可

      矩阵栈顶操作：OpenGL中常用的知识了。总所周知，对于矩阵的操作都是基于矩阵栈的栈顶来操作的，但有时候又希望使用变换前的矩阵，所以需要保存变换前的状态。具体做法：

      • PushMatrix：

        将当前的矩阵压入栈中，即保存当前矩阵的状态，此时栈最顶端的两个矩阵都是一样的，之后的矩阵的变换只修改栈顶，所以第二层的矩阵被保存下来了。

      • PopMatrix
        将栈顶的矩阵推出，此时原来第二层的的矩阵变成了栈顶矩阵，这就相当于恢复了之前保存的矩阵

        这里之所以要用栈顶操作是因为有可能一些操作可能会改变Camera的参数（比如用了LoadPixeMatrix()方法），所以需要将之前的状态保存下来。

        举个画矩形的栗子

        public GLTest : MonoBehaviour
{
  //绘制时需要使用的材质
  public Material drawMat;

  private Vector3 mouseStartPos;
  private Vector3 mouseMovePos;
  private bool isDraw = false;
  
  void Update()
  {
      if(Input.GetMouseButtonDown(0))
      {
          mouseStartPos = Input.mousePosition;
          isDraw = true;
      }
      else if(Input.GetMouseButtonUp(0))
      {
          isDraw = false;       
      }
      if(isDraw)
      {
          mouseMovePos = Input.mousePosition;
      }
  }
  void OnPostRender()
  {
      if(isDraw)
      {
          //设置材质通道为第一个
          drawMat.SetPass(0);

          //保存当前矩阵
          GL.PushMatrix();

              //设置以绘图模式为正交（2D用）
              GL.LoadOrtho();
              
              //矩形的四个顶点
              Vector3 left_top = mouseStartPos;
              Vector3 left_bottom = new Vector3(mouseStartPos.x, mouseMovePos.y);
              Vector3 right_top = new Vector3(mouseMovePos.x, mouseStartPos.y);
              Vector3 right_bottom = mouseMovePos;

              left_top = ScreenToGLPos(left_top);
              left_bottom = ScreenToGLPos(left_bottom);
              right_top = ScreenToGLPos(right_top);
              right_bottom = ScreenToGLPos(right_bottom);

              //画矩形
              GL.Begin(GL.LINES);
          
                  GL.Color(Color.red);
                  //左
                  GL.Vertex(left_top);
                  GL.Vertex(left_bottom);
                  //上
                  GL.Vertex(left_top);
                  GL.Vertex(right_top);
                  //右
                  GL.Vertex(right_top);
                  GL.Vertex(right_bottom);
                  //下
                  GL.Vertex(left_bottom);
                  GL.Vertex(right_bottom);

              GL.End();

          //恢复矩阵
          GL.PopMatrix();
      }
  }
  //转换到GL坐标，因为GL坐标是屏幕坐标在屏幕上所占的比例（0到1的值）
  Vector3 ScreenToGLPos(Vector3 pos)
  {
      return new Vector3(pos.x / Screen.width, pos.y / Screen.height);
  }
}

  • API

    • 静态变量

      • Begin模式

        变量 描述
        LINE_STRIP 连续绘制直线（三个顶点两条直线）
        LINES 以两两顶点绘制直线（四个顶点两条直线）
        QUADS 以四个顶点绘制一个四边形
        TRIANGLES 以三个顶点绘制一个三角形
        TRIANGLE_STRIP 连续绘制三角形

      • 其他变量

        变量 描述
        invertCulling 选择是 (true) 否 (false) 反转背面剔除
        modelview 获取或设置 modelview 矩阵
        sRGBWrite 控制是否在进行渲染时执行从线性到 sRGB 的颜色转换
        wireframe 是否应在线框中进行渲染

    • 静态函数

    函数 描述
    Begin 开始绘制 3D 图元
    End 结束绘制 3D 图元
    PopMatrix 从矩阵堆栈顶部恢复模型、视图和投影矩阵
    PushMatrix 将模型、视图和投影矩阵保存到矩阵堆栈顶部
    Vertex 提交一个顶点
    Vertex3 提交一个顶点，范围是左下角的(0,0,0)到右上角的(1,1,0）
    Clear 清除当前渲染缓冲区
    ClearWithSkybox 使用摄像机的天空盒清除当前渲染缓冲区
    Color 设置当前顶点颜色
    Flush 将驱动程序命令缓冲区中排队等待的命令发送到 GPU
    GetGPUProjectionMatrix 从摄像机的投影矩阵计算 GPU 投影矩阵
    InvalidateState 使内部缓存的渲染状态无效
    IssuePluginEvent 向本机代码插件发送用户定义的事件
    LoadIdentity 将标识加载到当前的模型和视图矩阵中
    LoadOrtho Helper 函数，用于设置正交投影
    LoadPixelMatrix 设置一个用于像素校正渲染的矩阵,
    LoadProjectionMatrix 将任意矩阵加载到当前投影矩阵。
    MultiTexCoord 对实际纹理 unit 设置当前纹理坐标 (v.x,v.y,v.z)
    MultiTexCoord2 为实际纹理 unit 设置当前纹理坐标 (x,y)
    MultiTexCoord3 对实际纹理 unit 设置当前纹理坐标 (x,y,z)
    MultMatrix 将当前模型矩阵设置为指定的矩阵
    RenderTargetBarrier 解析渲染目标，以便用于从该目标进行采样的后续运算
    TexCoord 为所有纹理单位均设置当前纹理坐标 (v.x,v.y,v.z)
    TexCoord2 为所有纹理单位均设置当前纹理坐标 (x,y)
    TexCoord3 为所有纹理单位均设置当前纹理坐标 (x,y,z)
    Viewport 设置渲染视口

    关于GL.LoadPixeMatrix()和GL.GL.LoadOrtho()：

    • GL.LoadPixeMatrix()：

      改变MVP矩阵，使得transform里的xy直接对应像素，Vertex3的xy值不需要除上Screen.width和Screen.height

    • GL.LoadOrtho()：

      正交视角，因为GL.Vertex3()取值范围从左下角的(0,0,0)到右上角的(1,1,0），所以需要除上Screen.width和Screen.height来转换到GL坐标
